# 테이블오더 서비스 요구사항

## Intent Analysis
- **User Request**: 테이블오더 서비스 MVP 개발
- **Request Type**: New Project (Greenfield)
- **Scope Estimate**: System-wide (Frontend + Backend + DB)
- **Complexity Estimate**: Complex (다중 매장, 실시간 통신, 인증, 파일 업로드)

---

## 기술 스택
- **Frontend**: React
- **Backend**: Python (FastAPI)
- **Database**: PostgreSQL
- **실시간 통신**: Server-Sent Events (SSE)
- **인증**: JWT (bcrypt 해싱)
- **배포**: Docker 컨테이너 기반
- **파일 저장**: 서버 로컬 파일 업로드

---

## Functional Requirements

### FR-01: 테이블 태블릿 자동 로그인
- 관리자가 매장 식별자, 테이블 번호, 비밀번호를 1회 설정
- 설정 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

### FR-02: 메뉴 조회
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세: 메뉴명, 가격, 설명, 이미지
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)
- 기본 화면으로 항상 표시

### FR-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 클라이언트 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

### FR-04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 성공 시: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 리다이렉트
- 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

### FR-05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)

### FR-06: 관리자 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반, 16시간 세션 유지
- bcrypt 비밀번호 해싱
- 매장당 1개 관리자 계정 (사전 등록)

### FR-07: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트 (2초 이내)
- 테이블별 그리드/카드 레이아웃
- 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
- 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료 (3단계)
- 신규 주문 시각적 강조

### FR-08: 테이블 관리
- 테이블 초기 설정 (번호, 비밀번호)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- 테이블 이용 완료 (세션 종료 → 주문 내역 과거 이력 이동 → 리셋)
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

### FR-09: 메뉴 관리
- 메뉴 CRUD (메뉴명, 가격, 설명, 카테고리, 이미지, 옵션)
- 이미지 파일 서버 업로드
- 메뉴 노출 순서 조정
- 옵션을 통한 사용자 커스텀 메뉴 제공
- 필수 필드 및 가격 범위 검증

### FR-10: 매장 관리
- 관리자 화면에서 매장 등록 기능
- 다중 매장 지원 (매장별 데이터 분리)

### FR-11 : 일별 매출 집계
- 업무 종료 시에 일별 매출 집계 자동화


### FR-12 : 주차 등록
- 주문 완료 후 무료 주차를 받기 위한 차량 번호 등록 기능
- 주문이 완료 되지 않았을 경우에는 노출하지 않음

## Non-Functional Requirements

### NFR-01: 성능
- SSE 주문 알림 2초 이내 전달
- 매장당 30개 이상 테이블 동시 지원

### NFR-02: 보안
- JWT 토큰 인증
- bcrypt 비밀번호 해싱
- 로그인 시도 제한

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적 카드 레이아웃
- 명확한 시각적 계층 구조

### NFR-04: 배포
- Docker 컨테이너 기반 배포
- Frontend/Backend/DB 컨테이너 분리

### NFR-05: 데이터
- 다중 매장 데이터 분리
- 주문 이력 영구 보관 (OrderHistory)
- 세션 ID 기반 주문 그룹화

## 제외 범위
- 결제 처리 (PG 연동, 영수증, 환불, 포인트/쿠폰)
- OAuth/SNS 로그인, 2FA
- 이미지 리사이징/최적화
- 푸시/SMS/이메일 알림
- 주방 전달 시스템, 재고 관리
- 직원 관리, 예약, 리뷰, 다국어
- 외부 연동 (배달, POS, 소셜 미디어)
