# User Stories

User Journey 기반으로 구성. 고객 여정 → 관리자 여정 순서.

## 고객 여정 (Customer Journey)

### US-01: 테이블 태블릿 자동 접속
**As a** 매장 고객 (김민수/박영희),
**I want to** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있기를,
**So that** 복잡한 절차 없이 즉시 주문을 시작할 수 있다.

**Acceptance Criteria:**
1. 태블릿 브라우저 접속 시 저장된 인증 정보로 자동 로그인된다
2. 자동 로그인 성공 시 메뉴 화면이 기본으로 표시된다
3. 자동 로그인 실패 시 매장 식별자/테이블 번호/비밀번호 입력 화면이 표시된다
4. 로그인 정보는 브라우저 로컬에 저장되어 재시작 시에도 유지된다
5. 세션은 16시간 동안 유지된다
6. **엣지 케이스**: 네트워크 오류 시 재시도 안내 메시지가 표시된다
7. **엣지 케이스**: 잘못된 인증 정보 입력 시 명확한 에러 메시지가 표시된다

**Personas**: 김민수, 박영희

---

### US-02: 메뉴 탐색
**As a** 매장 고객 (김민수/박영희),
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를,
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**Acceptance Criteria:**
1. 메뉴가 카테고리별로 분류되어 표시된다
2. 각 메뉴에 메뉴명, 가격, 설명, 이미지가 표시된다
3. 카드 형태 레이아웃으로 메뉴가 표시된다
4. 카테고리 간 빠른 이동이 가능하다
5. 모든 터치 버튼은 최소 44x44px 크기이다
6. **엣지 케이스**: 이미지 로딩 실패 시 기본 플레이스홀더 이미지가 표시된다
7. **엣지 케이스**: 메뉴가 없는 카테고리는 빈 상태 안내 메시지가 표시된다
8. **엣지 케이스**: 품절 메뉴는 시각적으로 구분되어 표시된다

**Personas**: 김민수, 박영희

---

### US-03: 장바구니 관리
**As a** 매장 고객 (김민수/박영희),
**I want to** 주문할 메뉴를 장바구니에 담고 수량을 조절하기를,
**So that** 주문 전에 원하는 메뉴와 수량을 정확히 확인할 수 있다.

**Acceptance Criteria:**
1. 메뉴 선택 시 장바구니에 추가된다
2. 장바구니에서 수량 증가/감소가 가능하다
3. 장바구니에서 메뉴 삭제가 가능하다
4. 총 금액이 실시간으로 계산되어 표시된다
5. 장바구니 비우기 기능이 있다
6. 페이지 새로고침 시에도 장바구니 내용이 유지된다 (로컬 저장)
7. **엣지 케이스**: 수량이 0이 되면 해당 메뉴가 장바구니에서 자동 삭제된다
8. **엣지 케이스**: 장바구니가 비어있을 때 안내 메시지가 표시된다
9. **엣지 케이스**: 동일 메뉴 재추가 시 수량이 증가한다

**Personas**: 김민수, 박영희

---

### US-04: 주문 생성
**As a** 매장 고객 (김민수/박영희),
**I want to** 장바구니의 메뉴를 최종 확인하고 주문을 확정하기를,
**So that** 정확한 주문이 매장에 전달된다.

**Acceptance Criteria:**
1. 주문 확정 전 주문 내역(메뉴명, 수량, 단가, 총 금액)을 최종 확인할 수 있다
2. 주문 확정 버튼 클릭 시 서버에 주문이 전송된다
3. 주문 성공 시 주문 번호가 표시된다
4. 주문 성공 후 장바구니가 자동으로 비워진다
5. 주문 성공 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다
6. 주문 정보에 매장 ID, 테이블 ID, 세션 ID가 포함된다
7. **엣지 케이스**: 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
8. **엣지 케이스**: 네트워크 오류 시 재시도 안내가 표시된다
9. **엣지 케이스**: 주문 확정 중 중복 클릭이 방지된다

**Personas**: 김민수, 박영희

---

### US-05: 주문 내역 조회
**As a** 매장 고객 (김민수/박영희),
**I want to** 현재 테이블에서 주문한 내역을 확인하기를,
**So that** 주문 상태와 총 금액을 파악할 수 있다.

**Acceptance Criteria:**
1. 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
2. 주문이 시간 순으로 정렬된다
3. 각 주문에 주문 번호, 시각, 메뉴/수량, 금액이 표시된다
4. 주문 상태(대기중/준비중/완료)가 표시된다
5. 이용 완료 처리된 주문은 표시되지 않는다
6. **엣지 케이스**: 주문이 없을 때 안내 메시지가 표시된다
7. **엣지 케이스**: 관리자가 주문을 삭제한 경우 목록에서 제거된다

**Personas**: 김민수, 박영희

---

### US-13: 무료 주차 차량 등록
**As a** 매장 고객 (김민수/박영희),
**I want to** 주문 완료 후 무료 주차 서비스를 위해 차량 번호를 등록하기를,
**So that** 별도의 절차 없이 무료 주차 혜택을 받을 수 있다.

**Acceptance Criteria:**
1. 주문이 1건 이상 완료(상태: 완료)된 경우에만 차량 등록 메뉴가 노출된다
2. 완료된 주문이 없으면 차량 등록 메뉴가 표시되지 않는다
3. 차량 번호를 입력하고 등록할 수 있다
4. 등록 성공 시 확인 메시지가 표시된다
5. 이미 등록된 차량 번호가 있으면 등록 정보가 표시되고 수정할 수 있다
6. 차량 번호는 현재 테이블 세션에 연결된다
7. 테이블 이용 완료 시 차량 등록 정보도 함께 이력으로 이동된다
8. **엣지 케이스**: 잘못된 차량 번호 형식 입력 시 검증 에러가 표시된다
9. **엣지 케이스**: 등록 실패 시 에러 메시지가 표시되고 재시도할 수 있다

**Personas**: 김민수, 박영희

## 관리자 여정 (Admin Journey)

### US-06: 관리자 로그인
**As a** 매장 운영자 (이정호),
**I want to** 매장 관리 시스템에 안전하게 로그인하기를,
**So that** 매장 주문과 메뉴를 관리할 수 있다.

**Acceptance Criteria:**
1. 매장 식별자 + 사용자명 + 비밀번호로 로그인한다
2. 로그인 성공 시 JWT 토큰이 발급된다
3. 세션이 16시간 동안 유지된다
4. 브라우저 새로고침 시에도 세션이 유지된다
5. 16시간 후 자동 로그아웃된다
6. 비밀번호는 bcrypt로 해싱되어 저장된다
7. **엣지 케이스**: 잘못된 자격 증명 시 에러 메시지가 표시된다
8. **엣지 케이스**: 로그인 시도 횟수가 제한된다

**Personas**: 이정호

---

### US-07: 매장 등록 및 관리
**As a** 매장 운영자 (이정호),
**I want to** 관리자 화면에서 매장을 등록하고 관리하기를,
**So that** 다중 매장을 운영할 수 있다.

**Acceptance Criteria:**
1. 새 매장을 등록할 수 있다
2. 매장별 데이터가 분리되어 관리된다
3. 매장 정보를 수정할 수 있다
4. **엣지 케이스**: 중복 매장 식별자 등록 시 에러 메시지가 표시된다
5. **엣지 케이스**: 필수 정보 누락 시 검증 에러가 표시된다

**Personas**: 이정호

---

### US-08: 실시간 주문 모니터링
**As a** 매장 운영자 (이정호),
**I want to** 들어오는 주문을 실시간으로 확인하기를,
**So that** 빠르게 주문을 처리할 수 있다.

**Acceptance Criteria:**
1. SSE 기반으로 신규 주문이 2초 이내에 화면에 표시된다
2. 테이블별 그리드/카드 레이아웃으로 표시된다
3. 각 테이블 카드에 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다
4. 신규 주문 시 시각적 강조(색상 변경, 애니메이션)가 적용된다
5. 테이블 카드 클릭 시 전체 주문 메뉴 목록 상세 보기가 표시된다
6. 테이블별 필터링이 가능하다
7. **엣지 케이스**: SSE 연결 끊김 시 자동 재연결 및 안내 메시지가 표시된다
8. **엣지 케이스**: 주문이 없는 테이블은 빈 상태로 표시된다

**Personas**: 이정호

---

### US-09: 주문 상태 변경
**As a** 매장 운영자 (이정호),
**I want to** 주문 상태를 단계별로 변경하기를,
**So that** 고객에게 주문 진행 상황을 알릴 수 있다.

**Acceptance Criteria:**
1. 주문 상태를 대기중 → 준비중 → 완료 순서로 변경할 수 있다
2. 상태 변경 시 즉시 반영된다
3. 변경된 상태가 고객 화면에도 반영된다
4. **엣지 케이스**: 역방향 상태 변경(완료 → 준비중)은 불가하다
5. **엣지 케이스**: 이미 완료된 주문의 상태는 변경할 수 없다

**Personas**: 이정호

---

### US-10: 테이블 관리
**As a** 매장 운영자 (이정호),
**I want to** 테이블 설정과 세션을 관리하기를,
**So that** 고객 회전을 효율적으로 처리할 수 있다.

**Acceptance Criteria:**
1. 테이블 번호와 비밀번호를 설정할 수 있다
2. 특정 주문을 삭제할 수 있다 (확인 팝업 후)
3. 주문 삭제 시 테이블 총 주문액이 재계산된다
4. 테이블 이용 완료 처리가 가능하다 (확인 팝업 후)
5. 이용 완료 시 해당 세션의 주문이 과거 이력으로 이동된다
6. 이용 완료 시 테이블 주문 목록과 총 주문액이 0으로 리셋된다
7. 새 고객이 이전 주문 없이 시작할 수 있다
8. **엣지 케이스**: 이용 완료 처리 실패 시 에러 메시지와 롤백이 수행된다
9. **엣지 케이스**: 주문이 없는 테이블도 이용 완료 처리가 가능하다

**Personas**: 이정호

---

### US-11: 과거 주문 내역 조회
**As a** 매장 운영자 (이정호),
**I want to** 테이블별 과거 주문 내역을 조회하기를,
**So that** 매출 현황과 이전 주문을 확인할 수 있다.

**Acceptance Criteria:**
1. "과거 내역" 버튼으로 과거 주문 목록에 접근할 수 있다
2. 테이블별 과거 주문이 시간 역순으로 표시된다
3. 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
4. 날짜 필터링이 가능하다
5. "닫기" 버튼으로 대시보드로 복귀할 수 있다
6. **엣지 케이스**: 과거 내역이 없을 때 안내 메시지가 표시된다

**Personas**: 이정호

---

### US-12: 메뉴 관리
**As a** 매장 운영자 (이정호),
**I want to** 메뉴를 등록, 수정, 삭제하기를,
**So that** 매장 메뉴를 최신 상태로 유지할 수 있다.

**Acceptance Criteria:**
1. 메뉴를 카테고리별로 조회할 수 있다
2. 새 메뉴를 등록할 수 있다 (메뉴명, 가격, 설명, 카테고리, 이미지)
3. 이미지 파일을 서버에 업로드할 수 있다
4. 기존 메뉴를 수정할 수 있다
5. 메뉴를 삭제할 수 있다
6. 메뉴 노출 순서를 조정할 수 있다
7. 필수 필드(메뉴명, 가격, 카테고리) 누락 시 검증 에러가 표시된다
8. 가격 범위 검증이 수행된다 (0 이하 불가)
9. **엣지 케이스**: 주문에 포함된 메뉴 삭제 시 확인 경고가 표시된다
10. **엣지 케이스**: 이미지 업로드 실패 시 에러 메시지가 표시되고 메뉴 등록은 이미지 없이 진행 가능하다
