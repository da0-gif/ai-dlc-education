# Business Rules - UOW-01

## BR-01: 인증 규칙

### BR-01-1: 테이블 로그인
- 매장 slug + 테이블 번호 + 비밀번호로 인증
- 인증 성공 시 세션 토큰 발급 (16시간 유효)
- 토큰은 localStorage에 저장, 자동 로그인에 사용
- 잘못된 인증 정보 시 에러 반환

### BR-01-2: 관리자 로그인
- 매장 slug + username + password로 인증
- password는 bcrypt로 검증
- 인증 성공 시 JWT 발급 (16시간 유효)
- 로그인 시도 제한: 5회 실패 시 15분 잠금

### BR-01-3: 세션 관리
- 테이블 세션: 첫 주문 시 자동 생성 (active session 없을 때)
- active session이 있으면 기존 세션 재사용
- 이용 완료 시 세션 종료 (is_active = false, ended_at 설정)

---

## BR-02: 주문 규칙

### BR-02-1: 주문 생성
- 장바구니 항목이 1개 이상이어야 함
- 각 항목의 수량 > 0
- total_amount = sum(unit_price × quantity)
- menu_name, unit_price는 주문 시점 스냅샷으로 저장
- 주문 생성 시 SSE 이벤트 발행

### BR-02-2: 주문 상태 전이
- 허용 전이: PENDING → PREPARING → COMPLETED
- 역방향 전이 불가 (COMPLETED → PREPARING ✗)
- 이미 COMPLETED인 주문은 상태 변경 불가
- 상태 변경 시 SSE 이벤트 발행

### BR-02-3: 주문 삭제
- 관리자만 삭제 가능
- 삭제 확인 필수 (프론트엔드)
- 삭제 시 테이블 총 주문액 재계산
- 삭제 시 SSE 이벤트 발행

### BR-02-4: 주문 번호
- 매장별 일별 auto-increment (매일 1번부터 시작)

---

## BR-03: 테이블 관리 규칙

### BR-03-1: 테이블 설정
- 테이블 번호: 매장 내 유일
- 비밀번호: 필수

### BR-03-2: 이용 완료
- 현재 세션의 모든 주문을 OrderHistory로 이동 (JSON 스냅샷)
- 세션 종료 (is_active = false)
- 주차 등록 정보도 이력에 포함
- 주문이 없는 테이블도 이용 완료 가능
- SSE 이벤트 발행

---

## BR-04: 메뉴 관리 규칙

### BR-04-1: 메뉴 CRUD
- 필수 필드: name, price, category_id
- price > 0 (0 이하 불가)
- 이미지: 선택사항, 업로드 실패 시 이미지 없이 등록 가능
- 주문에 포함된 메뉴 삭제 시 경고 표시 (삭제는 허용, 기존 주문의 스냅샷은 유지)

### BR-04-2: 카테고리 관리
- 관리자가 자유롭게 추가/수정/삭제
- 카테고리 삭제 시: 해당 카테고리의 메뉴가 있으면 삭제 불가
- sort_order로 노출 순서 관리

### BR-04-3: 메뉴 노출 순서
- sort_order 필드로 관리
- 관리자가 드래그 또는 순서 변경 API로 조정

---

## BR-05: 주차 등록 규칙

### BR-05-1: 등록 조건
- 현재 세션에서 상태가 COMPLETED인 주문이 1건 이상 존재해야 함
- 조건 미충족 시 주차 등록 메뉴 비노출

### BR-05-2: 차량 번호 검증
- 한국 차량 번호 형식 검증 (예: 12가3456, 서울12가3456)
- 세션당 1건만 등록 가능 (수정은 가능)

### BR-05-3: 이력 관리
- 테이블 이용 완료 시 OrderHistory에 주차 정보 포함

---

## BR-06: 매장 관리 규칙

### BR-06-1: 매장 등록
- slug(매장 식별자): 시스템 전체 유일
- 중복 slug 등록 시 에러

### BR-06-2: 관리자 계정
- 매장 등록 후 별도로 관리자 계정 생성
- 매장당 복수 관리자 가능

---

## BR-07: 일별 매출 집계 규칙

### BR-07-1: 일마감
- 관리자가 "일마감" 버튼 클릭 시 실행
- 해당 일자의 완료된 주문(COMPLETED) 금액 합산
- order_count: 완료된 주문 수
- 동일 날짜 중복 마감 시 기존 데이터 덮어쓰기
- 마감 시각(closed_at) 기록

---

## BR-08: SSE 이벤트 규칙

### BR-08-1: 이벤트 종류
- `order:created` - 신규 주문
- `order:status_changed` - 주문 상태 변경
- `order:deleted` - 주문 삭제
- `table:completed` - 테이블 이용 완료

### BR-08-2: 브로드캐스트 범위
- 매장별 구독자에게만 전송
- 연결 끊김 시 클라이언트 자동 재연결
